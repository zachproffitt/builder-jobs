Senior Security Engineer – Trusted Execution Environment (TEE / ARM TrustZone)
&nbsp;
Role Overview
We are looking for a security engineer to design and implement security-critical components running inside Trusted Execution Environments (TEE) such as ARM TrustZone.
In this role, you will develop Trusted Applications (TA) responsible for protecting DRM keys, secure media pipelines, device integrity verification, and hardware-backed security features for mobile and media platforms.
You will work closely with platform, operating system, and hardware teams to build robust security solutions protecting high-value digital content and platform services.
Responsibilities
Design and implement&nbsp;Trusted Applications (TA)&nbsp;running in&nbsp;ARM TrustZone / TEE environments
Develop secure communication between&nbsp;Normal World and Secure World
Implement&nbsp;secure key management and cryptographic operations inside TEE
Build&nbsp;DRM and secure media protection systems
Integrate TEE security services with&nbsp;Android or embedded platforms
Implement&nbsp;secure storage mechanisms (RPMB, secure file systems)
Design&nbsp;device attestation and hardware-backed security features
Perform&nbsp;threat modeling and attack surface analysis
Collaborate with OS, firmware, and hardware teams to integrate platform security features
Investigate and mitigate attacks such as
Investigate and mitigate attacks such as tampering, reverse engineering, and runtime manipulation
Required Qualifications
Strong programming experience in&nbsp;C/C++
Experience developing software for&nbsp;Trusted Execution Environments (TEE)&nbsp;or&nbsp;ARM TrustZone
Experience building&nbsp;Trusted Applications (TA)&nbsp;or secure platform services
Knowledge of&nbsp;secure monitor calls (SMC)&nbsp;and world switching mechanisms
Understanding of&nbsp;hardware root of trust, secure boot, and device attestation
Experience with&nbsp;secure key storage and applied cryptography
Experience working with Linux or embedded operating systems
Bachelor’s degree or equivalent experience in Security Engineering, or related field.
Preferred Qualifications
Experience with&nbsp;OP-TEE
Experience implementing&nbsp;DRM or secure media pipelines
Experience with&nbsp;secure storage (RPMB)
Experience with&nbsp;Android hardware-backed security (Keystore / KeyMint)
Familiarity with&nbsp;reverse engineering tools (Ghidra, IDA, Frida)
Experience in&nbsp;platform security, console security, or embedded device security
Knowledge of&nbsp;side-channel attacks and secure system design
